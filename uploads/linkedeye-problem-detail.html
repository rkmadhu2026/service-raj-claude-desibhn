Root Cause Analysis

Identified Root Cause

Problem: Connection leak in trading-api v2.4.3 causing progressive exhaustion of PostgreSQL connection pool during high-volume trading periods.

Contributing Factors:
  • No connection timeout configured in HikariCP pool
  • Insufficient connection pool monitoring alerts
  • Connection pool size (256) inadequate for peak load
  • Missing connection validation on checkout
